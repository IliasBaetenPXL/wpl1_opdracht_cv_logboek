Competenties

Ontwerper

Waar sta je momenteel?


Bij C# heb ik nog wat moeite, daarom ben ik er meer mee bezig
zodat ik geen achterstand creër.

Ik vind web een heel leuk vak en vind dat ik hier redelijk goed in ben.
Ik ben in mijn vrije tijd ook bezig om mijn eigen website te creëren voor mijn hobby.


Wat lukt je al goed?


Bij C# lukt het mij al heel snel om wat ik als design wil om te zetten in xeml-code
en zo het programma te maken hoe ik het mij inbeeld.

Bij HTML is het juist hetzelfde, ik weet wat ik wil en ik krijg het gelijk ik wil.
Dit doe ik doormiddel het van opzoeken op te zoeken op andere websites, in mijn cursus te kijken,
zelf op te zoeken en of zelf dingen aan te leren.


Wat lukt je nog niet?


Bij C# moet ik soms nog wel wat opzoeken als het gaat over het de code behind en soms
ook voor de xeml-code als ik iets van speciale elementen wil toevoegen

Bij web heb ik soms moeite met padding/margins aan te passen zodat
het mooi uitkomt als je het gaat schalen.

Programmeur

Waar sta je momenteel?


Bij C# probeer ik bezig te zijn met de oefeningen
van de les door deze opnieuw te maken want C# is een redelijk moeilijk vak voor me.

Deze pagina is mijn web voorbeeld.


Wat lukt je al goed?


Bij C# vind ik dat de logica mij redelijk goed lukt.

Web lukt mij het beste, ik weet wanneer ik een container moet gebruiken,
een div en hoe ik moet werken met de CSS animaties etc.


Wat lukt je nog niet?


C#: Binnen een bepaalde tijd werken is nog moeilijk

maar ik heb er vertrouwen in dat dit wel goed komt.

Web: De codes meer overzichtelijk schrijven zowel bij HTML als bij CSS,
dit komt zeker met ervaring.

Tester

Waar sta je momenteel?


Ik test mijn codes altijd wanneer ik een verandering heb uitgevoerd.
Dit kan zijn om de regel of om de 5 regels, zo ben ik zeker dat het werkt.
Al de fouten die ik vind, schrijf ik op en verbeter ik erna.
Dit doe ik zowel bij web als bij C#.


Wat lukt je al goed?


Bij C# weet ik waar ik op zoek moet gaan
om een breakpoint te zetten of een messagebox te plaatsen.
Bij web weet ik waar ik fouten moet opsporen met
behulp van de inspect tool op je web browser.


Wat lukt je nog niet?


C: ik maak nog niet veel gebruik van debugging tool.
Web: doordat ik soms een onoverzichtelijke code heb, wordt
het soms moeilijk om mijn fout op te sporen
